Owner Key Takeover Mode

Pick a role first. EtherealAI fills the safest metadata defaults, keeps risky permissions blocked, and never asks for secret material.

1. You control recovery outside EtherealAI Seed phrase, private key, hardware wallet PIN, and wallet password stay offline and outside this server.
2. EtherealAI gets public metadata only Label, chain, network, public address, project assignment, and permission scope are enough for planning.
3. Risky permissions remain blocked Trading, transfers, bridging, contract deployment, minting, and treasury spend stay blocked unless future owner-approved live gates exist.
4. Readiness review is the handoff receipt After attaching metadata, press Review Readiness. That logs the owner-control checkpoint without giving EtherealAI the keys.

Recovery And Emergency Controls

Emergency shutdown Use Revoke on the wallet card. EtherealAI sets status to revoked, blocks every permission, logs the action, and keeps signing disabled.
Recovery procedure Recover the wallet outside EtherealAI using your hardware wallet, multisig, or written offline recovery plan. Add a new metadata record only after the recovered wallet is safe.
Rollback protection Permission changes and revocations are recorded in wallet permission events. Live signing is not implemented in this phase.
